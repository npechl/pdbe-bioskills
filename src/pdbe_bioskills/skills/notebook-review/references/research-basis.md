# Research Basis

Notebook review borrows from:

- Kiro's spec review/checklist ideas for requirements, design, task, quality, and continuous improvement; source: https://kiro.dev/docs/specs/best-practices/
- nbdev's notebook-driven development emphasis that tests and documentation are first-class; source: https://pypi.org/project/nbdev/
- Diataxis documentation categories for tutorial, how-to, reference, and explanation; source: https://diataxis.fr/
- reproducible notebook best practices: tell a story for an audience, document the process, modularize code, record dependencies, use version control, share/explain data, and make notebooks read/run/explore friendly; source: https://notebooks.genepattern.org/best-practices/

InsightFold adaptation:

- Review must include scientific correctness and user interpretation, not just code quality.
- Execution passing is necessary but not sufficient.
- Public or AFDB/PDBe-facing notebooks should receive human/domain approval for scientific claims.

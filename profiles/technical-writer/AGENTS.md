# Purpose

Improve notebook pedagogy, README clarity, figure captions, limitations, and user-facing explanations.

# Responsibilities

- Make the notebook understandable to the target audience.
- Ensure each section explains purpose, interpretation, and limitations.
- Review captions, legends, warnings, empty states, and final summaries.
- Keep public-facing claims precise and technically honest.

# Skills To Invoke

- `$notebook-review`
- `$concept-to-prd` when reviewing PRD clarity

# Guardrails

- Do not make claims more impressive than the evidence supports.
- Do not hide caveats in dense footnotes or developer-only comments.

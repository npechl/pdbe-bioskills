# Scoping Question Bank

## Core Question

- What is the one biological, scientific, or workflow question this notebook is meant to answer?
- What would a user know or decide after running it that they could not know before?
- Why should this be a notebook before it becomes a production feature?

## User And Workflow

- Who is the primary user: structural biologist, bioinformatician, wet-lab scientist, curator, method developer, or product team?
- What does the user bring: accession, sequence, structure, mutation list, assay data, or uploaded files?
- What is the first runnable input cell?
- Which parameters should the user edit, and which should stay internal?
- What result should appear first to confirm the notebook is working?

## Scientific Context

- Which biological object or phenomenon is in scope?
- Which data source is authoritative?
- Which assumptions could make the interpretation invalid?
- Which outputs are descriptive versus decision-supporting?
- What must be phrased as RUO or exploratory?

## Implementation And Validation

- Which APIs, files, kernels, dependencies, and install cells are required?
- What fixture proves the happy path?
- What edge or negative fixture proves failure behavior?
- Which outputs need numerical tolerance, schema checks, visual checks, or human review?
- What provenance must be visible in notebook outputs?

## Lifecycle

- Is this a standing notebook, an integration candidate, or a discovery prototype?
- What evidence would justify promotion into AFDB/PDBe or the InsightFold database?
- What would make the project not worth continuing?
